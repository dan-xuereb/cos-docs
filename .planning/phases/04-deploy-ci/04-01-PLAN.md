  <action>
Create directory `cos-docs/deploy/` and write `cos-docs/deploy/nginx.conf` per RESEARCH §5. The config MUST:

- `listen 8080;` (non-privileged port so non-root UID 101 user can bind)
- `server_name _;`
- `root /usr/share/nginx/html;`
- `index index.html;`
- `location = /health { access_log off; return 200 "healthy\n"; add_header Content-Type text/plain; }` — k8s probe target, matches quant-dashboard precedent exactly.
- `location / { try_files $uri $uri/ $uri/index.html /404.html; }` — supports mkdocs Material's `use_directory_urls=True` which emits `site/architecture/index.html` and expects requests like `/architecture/`.
- `error_page 404 /404.html;` + `location = /404.html { internal; }` — serves MkDocs Material's auto-generated themed 404.
- `gzip on` with gzip_types covering text/html, text/css, text/plain, text/xml, application/javascript, application/json, application/xml, application/rss+xml, image/svg+xml. `gzip_comp_level 5`, `gzip_min_length 256`, `gzip_vary on`, `gzip_proxied any`.
- `add_header Cache-Control "no-cache";` at server level — Material emits non-content-hashed filenames (per RESEARCH §5), so conservative cache policy avoids stale JS/CSS after redeploys.
- Do NOT add SSL/TLS blocks (out of scope per CONTEXT.md D-05).
- Do NOT add proxy_pass anywhere (static-only serving; no upstream services — differs from quant-dashboard which proxies to ws-bridge).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cluster-internal → docs pod | NodePort 30081 exposes the pod to anything routable to 10.70.0.102; intentional per CONTEXT.md (internal-only network, no auth). |
| docker build context → image | `.dockerignore` is the only gate preventing secrets or dev artifacts from leaking into the image layer. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Elevation of Privilege | nginx container | mitigate | `USER 101` + `runAsNonRoot: true` (k8s securityContext set in Plan 04-02). Container cannot bind <1024 ports — enforced by `listen 8080`. |
| T-04-01-02 | Tampering | base image | mitigate | Pin `nginx:1.27-alpine` (not `:latest`). Rebuild surface reviewed at each minor version bump. |
| T-04-01-03 | Tampering | docker build context | mitigate | `.dockerignore` excludes `.planning/`, `.venv*`, `.git/`, `scripts/`. Prevents accidental secret/credential leakage into image layers. |
| T-04-01-04 | Information Disclosure | nginx server tokens | accept | Default `server_tokens on` leaks nginx version. Low value target on internal network; revisit if TLS + public exposure is ever added. |
| T-04-01-05 | Tampering | path traversal via nginx | mitigate | `try_files` only resolves under `root`; no `alias` directives; no proxy_pass upstreams. |
</threat_model>
