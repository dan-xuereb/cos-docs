  <action>
Create executable script at `/home/btc/github/cos-docs/scripts/emit-site-manifest.sh`. Per D-02 and D-10, emit JSON to stdout containing:

- `generated_at`: ISO-8601 UTC timestamp
- `cos_docs_sha`: value of `$GITHUB_SHA` if set, else `git -C /home/btc/github/cos-docs rev-parse HEAD`, else "unknown"
- `repos`: object mapping repo basename to commit SHA for each `/home/btc/github/*/` directory that has a `.git/` subdir, EXCLUDING `cos-docs` itself.

Script requirements:
- Shebang `#!/usr/bin/env bash` and `set -euo pipefail`.
- Accept optional `--workspace PATH` argument (default `/home/btc/github`).
- Non-git directories skipped silently (matches D-02 continue-on-failure spirit).
- If `git rev-parse HEAD` fails for a repo (detached w/ no commits, corrupt), emit `"unknown"` for that repo's SHA, do NOT abort.
- Output strictly valid JSON (pipeable into `jq -e .`). Use `jq -n` to build it so key ordering and escaping are correct.
- Only JSON on stdout; logs (if any) go to stderr.

Use this structure:

1. Parse args.
2. Resolve `cos_docs_sha` and `generated_at`.
3. Build a newline-delimited `name<TAB>sha` stream by iterating `"$WORKSPACE"/*/`, skipping `cos-docs` and non-git dirs.
4. Feed the stream into `jq -Rn --arg generated_at ... --arg cos_docs_sha ... '[inputs | split("\t") | {key: .[0], value: .[1]}] | from_entries | {generated_at: $generated_at, cos_docs_sha: $cos_docs_sha, repos: .}'`
5. If `jq` is unavailable, fall back to a hand-rolled JSON printer (log warning to stderr) — jq SHOULD always be present on the runner host; this fallback is belt-and-braces.

`chmod +x scripts/emit-site-manifest.sh` after writing.

./scripts/emit-site-manifest.sh | jq -e '.generated_at | test("^[0-9]{4}-[0-9]{2}-[0-9]{2}T")'

Per D-07 (scripted registration via `gh api`), create `cos-docs/scripts/install-runner.sh`. This is documentation-as-code — it is runnable on the Talos host but Task 3 is where the operator actually executes it.

Script MUST (in order):

1. Shebang `#!/usr/bin/env bash` and `set -euo pipefail`. Header comment citing D-01, D-07, D-10.
2. Define constants: `REPO=dan-xuereb/cos-docs`, `RUNNER_HOME=/home/btc/actions-runner`, `RUNNER_NAME=talos-cos-docs`, `RUNNER_LABELS=self-hosted,cos-docs,talos`.
3. Preconditions:
   - User is `btc` (`whoami` check).
   - `gh`, `jq`, `curl`, `tar` on PATH.
   - `gh auth status` succeeds.
   - `gh auth status 2>&1 | grep -q admin:repo_hook` (scope required for runner-registration-token endpoint). If missing, abort with "run gh auth refresh -s admin:repo_hook".
   - `/home/btc/github/cos-docs` directory exists.
4. Fetch latest runner release metadata: `gh api repos/actions/runner/releases/latest --jq '.assets[] | select(.name | test("^actions-runner-linux-x64-[0-9.]+\\.tar\\.gz$")) | .browser_download_url' | head -n1`. Extract version from the URL.
5. Prep `$RUNNER_HOME`: abort if `.runner` already exists (runner already registered; operator must remove explicitly).
6. Download tarball via `curl -fsSL -o runner.tar.gz "$asset_url"`.
7. Verify SHA256 by grepping the release body for a `64-hex + filename` line; compare to `sha256sum runner.tar.gz`. Abort on mismatch. Warn (not abort) if release body has no hash line.
8. `tar xzf runner.tar.gz && rm runner.tar.gz`.
9. Fetch registration token: `token=$(gh api -X POST repos/${REPO}/actions/runners/registration-token --jq .token)`. Abort if empty.
10. Register:
    ```
    ./config.sh --url https://github.com/${REPO} --token "$token" \
        --name "$RUNNER_NAME" --labels "$RUNNER_LABELS" \
        --work _work --unattended --replace
    ```
11. Install + start systemd service: `sudo ./svc.sh install btc && sudo ./svc.sh start`.
12. Verify: service name is `actions.runner.${REPO//\//-}.${RUNNER_NAME}.service`. Wait 2s, then `systemctl is-active <svc-name>` must return `active`; on failure dump `journalctl -u <svc-name> --no-pager -n 50` and exit 1.
13. Echo success + runner-status URL `https://github.com/${REPO}/settings/actions/runners`.

`chmod +x scripts/install-runner.sh`. Do NOT execute here (Task 3 handles that).

Validate syntax only: `bash -n scripts/install-runner.sh`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub.com → Talos host | Runner pulls jobs over HTTPS; outbound-only; no inbound ports opened. |
| Runner (user btc) → sibling repos | Runner has full read/write over /home/btc/github/* as user btc; git pull + build-all-api.sh operate with those privileges. |
| Runner → 10.70.0.30:5000 registry | docker push from runner host; plaintext insecure registry on internal VLAN. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | runner tarball | mitigate | SHA256 verification against GH release-body hash line; abort on mismatch. |
| T-04-03-02 | Elevation of Privilege | runner as btc | accept | Runner needs write access to /home/btc/github/* for git pull + build-all-api swap/restore. Scoping to a dedicated user beyond btc would require refactoring the workspace. Matches workspace norms. |
| T-04-03-03 | Spoofing | runner registration token | mitigate | Token fetched via gh api over HTTPS with user's authenticated gh CLI; scoped to single registration; expires in minutes. Never echoed to logs. |
| T-04-03-04 | Information Disclosure | site-manifest.json contents | accept | Manifest contains only repo basenames + commit SHAs. No secrets. Baked into image served on internal NodePort. |
| T-04-03-05 | Denial of Service | runner hung on stuck build | accept | workflow timeout-minutes: 45 caps per-job runtime (set in Plan 04-04). `concurrency` group serializes overlapping runs. |
</threat_model>
