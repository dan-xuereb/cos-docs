Create `cos-docs/scripts/scaffold.sh` (mode 0755, executable). Contents MUST include:

1. **Shebang and strict mode** (per workspace convention, matches quant-dashboard/build-images.sh):
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   ```

2. **Top-of-file usage comment** (per workspace convention):

   ```
   # scaffold.sh — Drop a working per-repo MkDocs Material setup into a target sibling repo.
   # Usage: scaffold.sh [--force] <path-to-target-repo>
   #   --force   Also overwrite user-owned docs/*.md files (default: leave existing docs/*.md untouched)
   # Implements decisions D-01..D-17 from .planning/phases/01-scaffold-template/01-CONTEXT.md
   ```

3. **Argument parsing** (per D-03, D-08):
   - Accept optional `--force` flag (sets `FORCE=1`, default `FORCE=0`)
   - Accept exactly one positional arg: `TARGET_REPO` (path to target repo)
   - On missing/extra args: print usage to stderr and `exit 2`
   - On non-existent path: print error and `exit 3`
   - Resolve `TARGET_REPO` to an absolute path via `cd "$TARGET_REPO" && pwd` (capture in subshell)

4. **Repo-type auto-detection** function `detect_repo_type()` (per D-09, D-10):
   - If `$TARGET_REPO/pyproject.toml` exists → echo `python` (D-10: python wins on mixed)
   - Elif `$TARGET_REPO/package.json` exists → echo `ts`
   - Else → echo `docs-only`

5. **Python package-name extraction** function `detect_python_package()` (per D-13):
   - Parse `[project].name` from `$TARGET_REPO/pyproject.toml`. Use a complete GNU awk snippet (no placeholder `{...}`):
     ```bash
     local raw_name
     raw_name=$(awk '/^\[project\]/{p=1;next} /^\[/{p=0} p && /^name[[:space:]]*=/{ match($0, /"([^"]+)"/, a); if (a[1] != "") { print a[1]; exit } }' "$TARGET_REPO/pyproject.toml" 2>/dev/null || true)
     ```
     (GNU awk required for the 3-arg `match()`. If the executor lacks gawk, an equivalent `sed -nE` fallback is acceptable: `sed -nE '/^\[project\]/,/^\[/{ s/^name[[:space:]]*=[[:space:]]*"([^"]+)".*/\1/p; }' | head -1`.)
   - Fallback if `raw_name` is empty: `raw_name=$(basename "$TARGET_REPO")`
   - **CRITICAL — normalize hyphens to underscores in BOTH paths** (per D-13; the resulting value is used as a Python import path in Plan 02's `::: <package_name>` mkdocstrings directive — `cos-core` is NOT a valid Python module name):
     ```bash
     local pkg_name="${raw_name//-/_}"
     echo "$pkg_name"
     ```
   - This MUST run regardless of whether the name came from pyproject.toml or the directory fallback. COS-Core's pyproject declares `name = "cos-core"`, but the importable package is `cos_core`; without this normalization Plan 02's `mkdocs build --strict` will fail.

6. **File-ownership write functions** (per D-05, D-06):
   - `write_user_owned <relative_path> <emit_function>`:
     - If file exists AND `FORCE=0`: print `[skip] $path (exists, use --force to overwrite)` to stderr, return 0
     - Else: call emit_function, write to path. Print `[write] $path` to stderr.
   - `write_scaffold_owned <relative_path> <emit_function>`:
     - If file exists: capture old content, write new content to a tmp file. Compare with `diff -u "$old" "$new"`; if diff output is non-empty, print it to stderr (per D-06). **If the diff is empty, suppress entirely — do NOT print a diff header or any "no changes" message.** Then move tmp → target.
     - If file does not exist: just write. Print `[write] $path` to stderr.
     - Use atomic write pattern: write to `${path}.tmp.$$` then `mv` (per workspace convention — atomic file writes from CLAUDE.md "Error Handling — Python — Atomic file writes" pattern, applied to bash here).

7. **Placeholder emit functions** (Plan 02 will replace bodies):
   - `emit_index_md()` — echo "TODO: Plan 02 will provide docs/index.md template"
   - `emit_architecture_md()` — echo "TODO: Plan 02 will provide docs/architecture.md template"
   - `emit_api_md()` — accept `$1` = package name; echo "TODO: Plan 02 will provide docs/api.md template for package $1"
   - `emit_mkdocs_yml()` — accept `$1` = site_name; echo "TODO: Plan 02 will provide mkdocs.yml template with site_name $1"
   - `emit_requirements_docs_txt()` — echo "TODO: Plan 02 will provide requirements-docs.txt with pinned versions"

8. **Main control flow** (`main()` function called at bottom):
   - Parse args
   - `REPO_TYPE=$(detect_repo_type)`
   - `SITE_NAME=$(basename "$TARGET_REPO")` (used as mkdocs `site_name`)
   - Print `[scaffold] target=$TARGET_REPO type=$REPO_TYPE force=$FORCE` to stderr
   - Ensure `$TARGET_REPO/docs/` directory exists (`mkdir -p`)
   - Always: `write_user_owned docs/index.md emit_index_md`
   - Always: `write_user_owned docs/architecture.md emit_architecture_md`
   - If `REPO_TYPE == python`:
     - `PKG=$(detect_python_package)`
     - `write_user_owned docs/api.md "emit_api_md $PKG"` (per D-11: only python repos)
   - Always: `write_scaffold_owned mkdocs.yml "emit_mkdocs_yml $SITE_NAME"`
   - Always: `write_scaffold_owned requirements-docs.txt emit_requirements_docs_txt`
   - Print `[scaffold] done` to stderr, exit 0

9. **Bash style notes**:
   - Use `${VAR}` brace form consistently
   - Quote all path expansions: `"$TARGET_REPO"`, `"$path"`
   - Use `local` for function-scoped variables
   - Use lowercase function names with underscores (snake_case)

Make file executable: `chmod +x cos-docs/scripts/scaffold.sh` after Write.

Per D-01, D-02: this is a SINGLE file. Do not create a `templates/` directory or split into helper scripts.
